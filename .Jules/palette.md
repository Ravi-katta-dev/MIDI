## 2023-10-24 - Accessibility for Dynamic Labels and Hidden Inputs
**Learning:** When generating DOM elements entirely via JS without JSX/frameworks, it's easy to forget `htmlFor` bindings and `tabIndex` attributes. A `display: none` file input inside a `<label>` relies completely on pointer events by default and is inaccessible to keyboard users unless the `<label>` itself receives `tabIndex="0"` and a `keydown` listener to trigger the underlying input.
**Action:** Always ensure any `<label>` dynamically created for a component has its `htmlFor` explicitly tied to the input's `id`. For visually hidden inputs (like `type="file"`), manually delegate keyboard focus and `Enter/Space` keydown events from the label to the hidden input.
