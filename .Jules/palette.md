## 2026-07-16 - Accessible Custom DOM Controls
**Learning:** This app heavily uses vanilla JS DOM construction (`DomHelper.js`) for forms. Setting properties via standard object assignment requires using `htmlFor` instead of `for`. Also, dynamically adding custom checkable components where container clicks trigger manual actions can interfere with native label-clicking behaviors unless we selectively filter out label clicks from the manual JS trigger.
**Action:** Always verify custom checkbox/radio click handlers when associating native `<label>` elements so they don't double-toggle, and use `htmlFor` when setting the "for" attribute dynamically in JS DOM properties.
